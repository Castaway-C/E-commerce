# AGENTS.md

这是一个学生实训项目，不完全追求生产级，但要结构清晰、可维护、方便协作。

## 协作约定

- 编码时注释要清晰，模块边界和接口约定要明确。
- 不要随意引入不必要的复杂依赖。
- 需求规格说明书不要改动，优先遵循它。
- 实现设计说明书是编码依据，如有冲突先指出问题。
- 普通用户和后台管理员不要混用。
- 要预留后续扩展空间。
- Python 环境为 3.12.10。
- 多人协作时，目录、命名、接口和文档要保持统一。

## 编码原则

- 先搭架构和公共约定，再写业务。
- 代码和注释要清楚，方便同学接手。
- 不要提前实现复杂业务流程。
- 如发现设计文档明显不一致或会影响实现，请先指出，再继续。
- 保持改动范围聚焦在本次任务内。
